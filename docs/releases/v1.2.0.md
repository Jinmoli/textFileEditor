## GitHub Release 正文

```markdown
## 更新亮点

- 新增编辑器内 `Ctrl/Cmd+F` 查找与替换面板，并完成中文界面适配。
- 完善 `xml / yaml / sql / properties / sh / bat / ps1` 等文本文件高亮体验。
- 按文件记忆只读和自动换行状态，减少多文件切换时的干扰。
- 改进 `Ctrl/Cmd+S` 快捷保存处理，只在当前文本编辑器上下文中接管快捷键。
- 新增 ASCII 手动编码选项，并保留旧编码文件重新读取入口。
- 补充 BRAT 安装说明、双语 README 和高亮能力矩阵文档。
- 优化生产构建配置，默认输出压缩产物，减小发布包体积。

## 适合谁升级

- 需要在 Obsidian 中频繁查看或编辑 `xml`、`yaml`、`sql`、`properties`、Shell、Batch 等文本文件的用户。
- 需要更顺手的查找/替换和快捷键体验的用户。
- 希望直接通过 BRAT 或 Release 安装插件的用户。

## 安装方式

- BRAT：使用仓库 `Jinmoli/textFileEditor`
- 手动安装：下载 Release 中的插件压缩包，解压后放入 `.obsidian/plugins/text-file-editor/`
```

## 发布前检查

- 确认 `manifest.json` 与 `package.json` 版本均为 `1.2.0`
- 确认 tag 使用 `v1.2.0`
- 推送 tag 后检查 GitHub Actions 的 Release 工作流是否成功
